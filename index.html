Introduction to SQLAlchemy ORM

ORM stands for Object Relational Mapper. ORM lets you work with a database using Python objects instead of raw SQL queries.

Instead of writing SQL:

SELECT * FROM users WHERE id = 1;

You can write Python code like:

user = session.get(User, 1)

SQLAlchemy has two main parts:

  1. Core – SQL expressions, closer to raw SQL
  2. ORM – Object-Oriented approach to interact with tables and rows

Model

Models are Python classes that represent database tables. Each model:

  • Maps to one table
  • Each attribute maps to one column
  • Each object maps to one row
class User(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    username = db.Column(db.String(50))

Column

A Column defines a field in a table, including its data type, constraints, and other attributes.

id = db.Column(db.Integer, primary_key=True)
username = db.Column(db.String(50), unique=True, nullable=False)

Primary Key

Primary key uniquely identifies each row in a table:

id = db.Column(db.Integer, primary_key=True)

Query

Query objects retrieve data using Python:

user = User.query.filter_by(username="john").first()

Session

The session tracks changes to objects and handles database communication:

db.session.add(new_user)
db.session.commit()

Commit

Commits changes in the current session to the database:

db.session.commit()

Rollback

Reverts uncommitted changes in case of errors:

db.session.rollback()

Engine

The engine connects SQLAlchemy to the database:

app.config['SQLALCHEMY_DATABASE_URI'] = 'mysql+pymysql://root:@localhost/login_db'

Metadata

Metadata stores information about tables, columns, and constraints:

with app.app_context():
    db.create_all()
